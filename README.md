# 🇰🇷 Korean Tokenizer Benchmark: Subword vs. Morphological
> **뉴스 코퍼스 기반 한국어 토크나이저 성능 및 효율성 비교 분석 프로젝트**

---

## 1. 프로젝트 개요
* **목적**: 한국어 자연어 처리(NLP) 최적화를 위한 토크나이징 알고리즘별 정량적 비교 분석
* **핵심 과제**: 
    * 학습 데이터 규모(10%, 50%, 100%)에 따른 Subword 알고리즘(BPE, WordPiece)의 성능 변화 추이 분석
    * 주요 한국어 형태소 분석기(Mecab, Okt, Kkma) 간 처리 속도 및 분절 정밀도 벤치마크
* **핵심 지표 (Metrics)**: 
    * **Fertility**: 원문 단어 대비 생성 토큰 비율 (분절 효율성 및 시퀀스 길이 영향도 평가)
    * **UNK Rate**: 어휘 사전 미포함 단어(Unknown Token) 발생 빈도 (모델의 일반화 성능 평가)
    * **Inference Speed**: 문장당 평균 처리 속도 (ms)
---

## 2. 데이터 통계
* **데이터원**: 네이버 뉴스 API (경제, 사회, IT 분야)
* **데이터 규모**: 약 17,327개 문장 (파일 크기 4.66MB)
* **어휘 구성**: 총 단어 수(공백 기준) 458,165개 / 고유 어휘 수 114,998개

## 5. 핵심 결론
* **데이터 규모와 일반화 능력**: 데이터 증가에 따른 UNK Rate의 급격한 감소(최대 19배) 확인. 한국어 뉴스 데이터셋 기준 **WordPiece**가 BPE 대비 적은 데이터로도 높은 어휘 커버리지를 확보함.
* **알고리즘별 최적 활용 방안**:
    * **실시간 서비스**: 형태소 분석의 정확도와 속도 균형이 뛰어난 **Mecab** 권장.
    * **딥러닝 기반 언어 모델**: 미등록어 대응이 유연하고 시퀀스 정보 보존에 유리한 **WordPiece** 권장.
    * **가독성 중요**: **Okt**는 가장 낮은 Fertility(3.25)를 보여 텍스트 요약이나 키워드 추출 등 가독성이 중요한 태스크에 강점이 있음.
